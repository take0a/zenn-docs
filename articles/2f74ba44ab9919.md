# やってみた
## [インストール](https://docs.evidence.dev/install-evidence/)

VSCode拡張もありますが、やはり生ですよね。ということでコマンドラインからインストールしました。事前に nvm で node v24.15.0 を入れてあります。
```cmd
npx degit evidence-dev/template my-project
cd my-project
npm install
npm run sources
npm run dev
```

勝手にブラウザが立ち上がって、こんな感じになりました。
![](/images/2f74ba44ab9919/evidence.png)

## [新しいページを作る](https://docs.evidence.dev/build-your-first-app/#2-add-a-new-page)

見出しのリンクの本家のドキュメントのとおり、新しいページを作ります。

## Hello Evidence

## [クエリを書く](https://docs.evidence.dev/build-your-first-app/#5-set-up-a-source-query)
上の見出しに書いてある SQL は、既にあります。
```sql: sources/needful_things/order.sql
select * from orders
```

これが、`pages/index.md` 内で `needful_things.orders` として使えるよ。これを使えるようにする「おまじない」が `npm run sources` で、これをやると、本物のデータソースから Parquet 形式のキャッシュを作るというのが仕組みのようです。
データソースの SQL 方言で SQL ファイルにクエリを書いて、そのキャッシュに対するクエリは、マークダウンのコードブロック内に DuckDB 方言で書く。
````
```sql categories
  select
      category
  from needful_things.orders
  group by category
```
````
そのコードブロックの名前がデータセットとして、マークダウン内のコンポーネントタグ内で使用する。
```
<Dropdown data={categories} name=category value=category>
    <DropdownOption value="%" valueLabel="All Categories"/>
</Dropdown>
```
なので、Evidence 用に知る必要があるのは、[SQL](https://docs.evidence.dev/core-concepts/queries/) と [コンポーネント](https://docs.evidence.dev/components/all-components/) ということになります。

SQL については、DuckDB 方言とのことですが、データの受け渡しが SQL 間、コンポーネントとの間、ソースSQLなど独自の部分があるはずなので勉強しましょう。

コンポーネントは、結構ありますが、独自コンポーネントも作れるようなのでコダワル方への対応もできそうです。
